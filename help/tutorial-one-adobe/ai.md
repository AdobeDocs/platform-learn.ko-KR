# 개요 - Agentic AI Tech Labs

![기술 내부자](./assets/images/techinsiders.png){width="50px" align="left"}

## 개요 - 진행 중인 작업

[1.1 Agent Orchestrator](./modules/agents/module1.1/agentorchestrator.md)

**목표**

Adobe Experience Platform 에이전트 및 Agent Orchestrator을 사용하여 다음을 수행하는 방법에 대해 알아봅니다.

- 구매 트렌드 분석
- 성향이 높은 대상자 식별
- 여정 성능 확인
- CitiSignal 파이버 최대 롤아웃을 위한 새 여정 만들기

[1.4 Brand Concierge](./modules/agents/module1.4/brandconcierge.md)

**목표**

Brand Concierge은 브랜드가 웹 사이트 방문자와 참여하는 방식을 혁신하는 AI 기반의 디지털 동반자입니다. 일반적인 챗봇과 달리 Brand Concierge은 각 방문자의 의도에 맞게 개인화된 대화 경험을 제공합니다. 방문자가 제품을 검색하고 옵션을 비교하며 즉각적인 답변을 얻고 실시간으로 가이드 추천을 받을 수 있도록 지원합니다. 플랫폼은 B2C와 B2B를 모두 제공하며, 브랜드 음성, 콘텐츠 무결성 및 규정 준수를 유지하면서 모든 디지털 채널에서 브랜드를 지능적으로 확장하는 역할을 합니다.

이 연습에서는 다음 방법을 배웁니다.

- Adobe Experience Platform 샌드박스에서 Brand Concierge 인스턴스 구성
- AEM CS/EDS 웹 사이트에서 Brand Concierge 구현

[1.5 Analytics 및 에이전트](./modules/agents/module1.5/analyticsagents.md)

**목표**

데이터 분석가, AI 개발자 또는 AI 애플리케이션 설계자는 보고서 생성, 외부 에이전트를 사용한 분석 예약 등과 같은 보고 작업을 자동화하는 방법을 배울 수 있습니다. 새로운 캠페인 데이터, 대상자 데이터 또는 성능 데이터를 에이전트 워크플로우로 가져오는 방법을 배웁니다.

이 연습에서는 다음 방법을 배웁니다.

- ChatGPT 및/또는 Claude.ai를 **Customer Journey Analytics**&#x200B;에 연결하고 데이터 분석 작업을 수행합니다
- ChatGPT 및/또는 Claude.ai를 **Adobe Analytics**&#x200B;에 연결하고 데이터 분석 작업을 수행합니다

[1.6 AEM 및 에이전트](./modules/agents/module1.6/aemagents.md){target="_blank"}

**목표**

이제 Adobe Experience Manager에는 여러 특별히 제작된 에이전트가 포함되어 있으며, 각 에이전트는 역사적으로 엄청난 수작업을 필요로 했던 작업을 수행하도록 설계되었습니다. 이들은 일반적인 AI 어시스턴트가 아니며, AEM을 깊이 있게 이해하고 콘텐츠, 코드, 에셋, 거버넌스 및 최적화 전반에 걸쳐 작동하는 도메인 교육을 받은 에이전트입니다.

- 업데이트, 콘텐츠 변경 및 전체 사이트 마이그레이션까지 가속화하는 **Experience Production Agent**.
- **거버넌스 에이전트**&#x200B;에서 브랜드, 권한 및 규정 준수 규칙을 자동으로 적용합니다.
- **검색 에이전트**&#x200B;는 AI 기반 검색을 위한 콘텐츠를 준비하고 지능형 전략가로 작동합니다.
- **콘텐츠 최적화 에이전트**&#x200B;은(는) 즉시 성능 준비가 완료된 채널별 자산 변형을 만듭니다.
- **Development Agent**&#x200B;는 AI 지원 문제 해결 및 성능 조정을 통해 개발자를 가속화합니다.

이 연습에서는 사용자 지정 MCP 서버 설정을 통해 AI Assistant와 Cursor를 모두 사용하여 이러한 에이전트를 사용하는 방법에 대해 알아봅니다.

[1.7 Adobe Commerce용 지능형 개발자 도구](./modules/agents/module1.7/aiassisteddev.md)

**목표**

이 모듈에서는 Cursor와 같은 지능형 개발자 도구를 사용하여 Adobe Commerce as a Cloud Service 환경에 대한 확장을 개발합니다. 해당 확장의 목표는 들어오는 주문 이벤트를 서드파티 엔드포인트로 전달하는 것입니다. Adobe Commerce as a Cloud Service의 이벤트 전달은 Adobe I/O App Builder, Adobe I/O Events 및 Adobe I/O Runtime을 사용합니다. 이러한 모든 서비스의 구성은 Cursor의 도움을 받습니다.
